# 使用手册

## 一、每日选股

运行主程序，获取当日短线/波段推荐：

```bash
python main.py
```

输出文件：
- `reports/report_YYYYMMDD.txt` — 每日选股报告
- `logs/trading_log_YYYYMMDD.log` — 运行日志

---

## 二、批量个股分析

对自选股列表进行 AI 深度分析（技术面 + 资金面 + 基本面）。

### 准备股票列表

编辑 `watchlist.txt`，每行一个代码：

```
# 注释行（#开头）会被忽略
600519
000858.SZ
601318 中国平安   # 名称会被忽略，只取代码
300750
```

支持 6 位纯代码或带后缀（如 `600519.SH`）。

### 运行分析

```bash
python main.py analyze watchlist.txt
# 或指定其他文件
python main.py analyze my_stocks.txt
```

### 查看报告

```
reports/batch_analysis_YYYYMMDD.txt
```

报告包含：综合评级（0-100分）、走势预测、目标价、止损价、买入时机、技术/资金/基本面分析、风险提示。

**评级说明：**
- 强烈推荐（90-100）：三维俱佳，趋势明确
- 推荐（70-89）：至少两维良好
- 中性（50-69）：建议观望
- 不推荐（30-49）：至少一维较差
- 强烈不推荐（0-29）：多维表现差

**注意：** 建议每次不超过 20 只，避免 API 超时；每只约需 3-5 秒。

## 三、回测

验证选股策略的历史表现，与实盘选股逻辑完全一致。支持**在线**和**离线**两种模式。

```bash
# 在线模式（直接调 Tushare，受网络速率限制）
python backtest_v2.py --start 20250101 --end 20251231

# 离线模式（从本地缓存读取，需先下载数据，速度快10-20倍）
python backtest_v2.py --start 20250101 --end 20251231 --offline
```

**完整参数说明：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--start` | 90天前 | 回测开始日期 YYYYMMDD |
| `--end` | 昨天 | 回测结束日期 YYYYMMDD |
| `--hold` | 5 | 持有天数 |
| `--topn` | 3 | 每日最多买入前N只 |
| `--fallback-stop` | -7.0 | 兜底止损线（%，技术位失效时使用） |
| `--fallback-profit` | 10.0 | 兜底止盈线（%，技术位失效时使用） |
| `--trailing-stop` | 5.0 | 移动止损回撤幅度（%，盈利≥3%后激活） |
| `--min-open-ratio` | 0.995 | 次日开盘确认比例：T+1开盘/T收盘 须≥此值才买入；低开超0.5%则跳过，过滤跳空低开后被动持有的亏损；设为 `0` 可关闭此过滤 |
| `--no-timing` | 关闭 | 忽略大盘择时，纯验证选股逻辑 |
| `--offline` | 关闭 | 启用离线模式（从本地 Parquet 读取） |
| `--cache-dir` | data/cache | 离线缓存目录路径 |

**对比回测技巧：**

```bash
# 开启开盘确认（默认）
python backtest_v2.py --start 20250101 --end 20251231 --offline

# 关闭开盘确认（纯验证选股逻辑，不过滤低开）
python backtest_v2.py --start 20250101 --end 20251231 --offline --min-open-ratio 0

# 收紧开盘确认（只买高开的票）
python backtest_v2.py --start 20250101 --end 20251231 --offline --min-open-ratio 1.005
```

输出文件：`backtest_results/` 目录下的 CSV（逐笔明细）和 JSON（汇总指标）。

指标包含：胜率、盈亏比、夏普比率、最大回撤、沪深300超额收益。

**说明：** 默认启用大盘择时，市场主跌浪期间自动空仓（与实盘一致）。加 `--no-timing` 可单独测试选股胜率。

## 四、离线回测（推荐）

在线回测每个交易日要调用多次 Tushare API，速度慢且容易因网络波动中断。  
**离线模式**将数据预先下载到本地 Parquet 文件，回测时完全脱离网络，速度提升 10-20 倍。

### 第一步：预下载数据

```bash
# 下载指定区间的所有数据（含财务指标）
python data_downloader.py --start 20250101 --end 20251231

# 跳过财务数据（更快，适合纯技术面回测）
python data_downloader.py --start 20250101 --end 20251231 --skip-financial

# 强制重新下载（更新已有数据时使用）
python data_downloader.py --start 20250101 --end 20251231 --force
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `--start` | 回测开始日期（脚本自动额外下载前120天数据供MA60计算） |
| `--end` | 回测结束日期 |
| `--force` | 强制覆盖已存在的文件 |
| `--skip-financial` | 跳过 fina_indicator 和 income 下载（适合只测技术面策略） |

**断点续传：** 已下载的文件自动跳过，中断后重新运行会从断点继续，不会重复下载。

**下载时间参考：**

| 回测区间 | 交易日数 | 约需时间 |
|---------|---------|---------|
| 1个月 | ~22天 | 5-10 分钟 |
| 1季度 | ~65天 | 15-25 分钟 |
| 1年 | ~245天 | 1-1.5 小时 |
| 财务数据（额外） | 全量约5000只 | 15-30 分钟 |

### 第二步：离线回测

```bash
python backtest_v2.py --start 20250101 --end 20251231 --offline
```

启动时会自动打印数据完整性报告：
```
[LocalDataProxy] 数据覆盖报告
  缓存目录：E:\代码项目\stock\data\cache
  trade_cal           :   1520 行  ✓
  stock_basic         :   5231 行  ✓
  fina_indicator      :   5231 行  ✓
  income              :  20924 行  ✓
  daily               :  245 天  [20241101 ~ 20251231]  ✓
  daily_basic         :  245 天  [20241101 ~ 20251231]  ✓
  moneyflow           :  245 天  [20241101 ~ 20251231]  ✓
  index_daily         :  245 天  [20241101 ~ 20251231]  ✓
```

有 `✗` 标记的项目说明数据缺失，需重新运行 `data_downloader.py`。

### 缓存目录结构

```
data/cache/
├── trade_cal.parquet          # 交易日历
├── stock_basic.parquet        # A股基础信息
├── share_float.parquet        # 限售股解禁数据
├── stk_holdertrade.parquet    # 股东增减持数据
├── fina_indicator.parquet     # 财务指标（ROE、负债率，每股最新期）
├── income.parquet             # 利润表（营收，每股近4期，用于同比）
├── daily/         YYYYMMDD.parquet  ← A股全市场日线（OHLCV）
├── daily_basic/   YYYYMMDD.parquet  ← 换手率、量比
├── moneyflow/     YYYYMMDD.parquet  ← 主力资金净流入
└── index_daily/   YYYYMMDD.parquet  ← 大盘指数 + 28个申万一级行业指数
```

## 六、止盈止损设定

系统采用**纯技术分析**确定目标价和止损价，不使用固定百分比。

### 目标价（压力位优先级）

1. 前高（`high20`，近 20 日最高价）
2. 均线压力（MA5 → MA10 → MA20 → MA60，取最近压力位）
3. 整数关口（100、50、200 等心理价位）

资金强势（量比 > 2.5 + 净流入 > 5000 万）→ 取第二压力位；中等 → 第一压力位；弱势 → 当前价。

### 止损价（支撑位优先级）

1. 最近均线支撑（MA5 / MA10 / MA20 / MA60）
2. 前低（`low20`，近 20 日最低价）
3. 整数关口

波动率高（> 5%）→ 支撑位下方 3-5%；正常（3-5%）→ 下方 2-3%；低（< 3%）→ 下方 1-2%。

**示例：**
```
当前价 100 元，MA5 = 98，前高 = 110，资金中等
→ 目标价：108-110 元（前高附近）
→ 止损价：95-96 元（MA5 下方 2-3%）
```

## 七、AI 模型选择

在 `config.py` 中修改 `AI_CONFIG` 的 `model` 字段切换模型。

| 模型 | 配置 `model` | 单次成本 | 能力 | 推荐 |
|------|-------------|---------|------|------|
| 通义千问 Turbo | `qwen-turbo` | ~0.02 元 | 60 分 | 预算极限时 |
| 通义千问 Plus | `qwen-plus` | ~0.20 元 | 75 分 | 无需换平台 |
| DeepSeek | `deepseek-chat` | ~0.05 元 | 85 分 | **推荐** |
| DeepSeek Reasoner | `deepseek-reasoner` | ~0.25 元 | 90 分 | 重要决策 |

**切换到 DeepSeek：**

```python
# config.py
AI_CONFIG = {
    "api_key": os.environ.get("DEEPSEEK_API_KEY", "你的Key"),
    "base_url": "https://api.deepseek.com/v1",
    "model": "deepseek-chat",
    "timeout": 60,
    "temperature": 0.1,
    "max_tokens": 6000
}
```

注册地址：https://platform.deepseek.com/（国内可直接访问，无需翻墙）

## 八、常见问题

**Q: 分析失败怎么办？**  
查看 `logs/trading_log_YYYYMMDD.log`。常见原因：API Key 未配置、网络问题、Tushare 积分不足。

**Q: 某只股票失败会影响其他股票吗？**  
不会。程序跳过失败的股票继续处理。

**Q: 回测交易数太少？**  
大概率是大盘择时触发空仓。加 `--no-timing` 禁用择时可测试更多交易。

**Q: 可以自定义 AI 分析维度吗？**  
修改 `ai_prompts.py` 中的提示词。

**Q: 在线回测经常中途断掉怎么办？**  
推荐改用离线模式：先用 `data_downloader.py` 把数据下载到本地，再加 `--offline` 参数运行回测。离线模式不依赖网络，也不受 Tushare 限速影响，速度快10-20倍。

**Q: 离线回测结果和在线回测一样吗？**  
完全一致。`local_data_proxy.py` 的接口签名与 Tushare pro 对象完全相同，底层调用的是同一套选股函数（`run_daily_selection`），数据来源不同但逻辑完全复用。

**Q: data_downloader.py 下载中断了，要重头再来吗？**  
不用。脚本支持断点续传：已存在的 Parquet 文件会自动跳过，重新运行会从中断处继续。

**Q: 离线数据占多大磁盘空间？**  
大约 1年数据（245个交易日）占 500MB-1GB，含财务数据约 1-1.5GB。按 Parquet + Snappy 压缩，比原始 CSV 小约 60-70%。

**Q: 下载完数据之后回测区间能随意调整吗？**  
可以，只要回测区间在已下载数据的范围内就行。注意：回测 `--start` 日期之前需要有120天的缓冲数据（供MA60计算），`data_downloader.py` 已自动处理这部分。

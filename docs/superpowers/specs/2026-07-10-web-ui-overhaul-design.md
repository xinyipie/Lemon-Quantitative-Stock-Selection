# 全站前端体验重构设计

## 目标

在不引入前端框架、不改变选股策略、不增加交易执行能力的前提下，一次性提升现有研究看板的决策效率、移动端可用性、页面性能、数据表达准确性和全站视觉一致性。

## 已评估方案

### 方案 A：只做样式修补

修复未定义类、响应式和表格溢出，不调整页面结构。风险低，但不能解决首页重复表达、市场雷达过长、短线与长线页面体积过大的根因。

### 方案 B：渐进式全站重构（采用）

保留 FastAPI、Jinja2 和现有服务层，在模板、共享样式和少量路由上下文中完成重构。通过共享组件、服务端分页、折叠分区和持久化雷达缓存改善体验，同时保持现有 URL 与核心数据接口兼容。

### 方案 C：改造成前后端分离单页应用

交互上限最高，但会引入构建链、API 重构和双端维护成本，不符合当前本地研究工具的规模，也会扩大部署风险。

## 全局设计

### 页面外壳与导航

- 桌面端侧栏固定在视口内，长页面滚动时仍能访问导航。
- 1080px 以下显示紧凑移动导航按钮，导航可展开和收起，不再把七个入口纵向铺满首屏。
- 主内容区设置合理最大宽度；研究表格仍可横向滚动，正文与卡片不在超宽屏无限拉伸。
- 所有页面统一使用 `terminal-header`、`panel-head`、`panel-title-row` 和标准标签色；补齐当前缺失的 `page-header`、`panel-title-row`、`success-tag` 样式。
- `.status-strip` 使用自适应列数，不再固定六列；中等宽度保持两至三列，手机端单列。

### 可访问性与表单

- 搜索和日期筛选增加可读标签；视觉上可使用隐藏标签，但不能只依赖 placeholder。
- 日期字段改为 `type="date"`，服务端同时接受 `YYYY-MM-DD` 与现有 `YYYYMMDD`。
- 增加 `:focus-visible` 样式、导航 `aria-label`、移动菜单 `aria-expanded`、表格区域说明。
- 长表格使用统一 `.table-shell`，包含横向滚动、粘性表头和边缘提示。

### 性能与页面体积

- 短线历史表默认每页 50 条，最多保留现有 300 条筛选上限；统计口径明确标注为“当前筛选样本，最多 300 条”。
- 长线历史样本默认每页 50 条，保留最多 1000 条筛选上限；当前池、生命周期和审计总览不受分页影响。
- 市场雷达将完整页面数据持久化为本地缓存。内存缓存失效或服务重启时优先读取与查询日期匹配的磁盘缓存；更新任务完成后生成新缓存。缓存读取失败时安全回退到实时计算。
- 静态样式版本改为新的发布版本，避免浏览器继续使用 7 月 1 日的旧 CSS。

## 页面设计

### 工作台

首屏顺序调整为：页面标题与搜索、数据状态、今日决策区、同步提示。强推荐或观察候选存在时展示对应卡片；两者都为空时合并为单个“今日暂不出手”状态，避免四个模块重复表达空信号。完整重算保留，但提交前需要二次确认并明确预计耗时。

AI 摘要、最近解释、实盘历史、回测预览和长线池摘要作为二级信息，默认保持紧凑，必要时使用折叠区。

### 短线复盘

保留强推荐、观察候选和真实推送历史的分层。历史 Top3 信号增加分页导航，表头保持可见。页面明确区分：

- 当前筛选结果总数；
- 当前页显示数量；
- 绩效统计所使用的样本上限和已满期样本数。

策略命名统一为“v9 底层评分 + v39 强推荐层”，避免页面同时出现 v9、v39、`short_v9_final` 而没有解释。

### 热门龙头

保留优先关注、谨慎观察、研究样本三栏。桌面端每栏默认展示前三张卡片，更多内容放入展开区，取消三个互不关联的固定高度滚动窗。补齐头部、标题行和成功标签样式。

### 长线池

页面分为“当前池”“生命周期”“历史验证”三个锚点页签。当前结论、Elite/Watch 和最近运行优先展示；完整运行日志与长线样本明细默认折叠。历史样本支持服务端页码参数，筛选变化时回到第一页。

### 市场雷达

顶部增加固定的页面内导航：研究简报、关键事件、主线判断、策略共振、消息概念、板块候选、风险提醒。首屏只保留研究员简报、风险阻断和关键事件，其余模块按研究顺序排列并允许折叠。

雷达 GET 页面不再承担不必要的快照写入；快照与页面缓存由更新流程或明确的刷新动作生成。历史日期查询使用独立日期控件并清晰显示当前数据日。

### 单股体检

缺失数值统一显示 `-`，真实的零值必须显示为 `0`，禁止使用 `value or 0`、`value or '-'` 混淆两种状态。顶部加入由现有日线数据生成的轻量价格趋势图；不引入外部图表依赖，使用内联 SVG，展示收盘价、MA20 和 MA60。

历史信号表默认显示核心列，可信度、质量、过程和 AI 状态合并为“复盘标签”；详细字段可展开查看。

### 数据状态

统一页面头部样式，增加网页同步入口和数据落后天数提示。删除“页面不会自动拉取数据”的过时描述；CLI 命令保留在“高级操作”折叠区。

### AI 解释

将 `fallback`、`short_v9_final` 等内部值转换为用户可读标签。页面明确分为“信号当时已知信息”和“事后验证”，避免后见之明混入当时判断。“轻仓”“考虑买入”等交易动作措辞改为纯观察条件。重新生成使用 POST 按钮并展示处理中状态，避免带副作用的 GET 链接。

## 数据与接口

- 现有页面 URL 保持不变。
- `/signals` 新增 `page` 参数，页码从 1 开始，固定每页 50 条。
- `/longterm` 新增 `page` 参数，页码从 1 开始，固定每页 50 条。
- `/explain/signal/{trade_date}/{ts_code}/refresh` 新增 POST 路由；旧的 `?refresh=true` 暂时保留兼容，但页面不再生成该链接。
- 单股趋势图数据由服务层返回最近最多 120 个交易日的日期、收盘、MA20、MA60；无数据时展示明确空状态。
- 分页信息统一为 `page_info = {page, page_size, total, total_pages, start_index, end_index}`。

## 错误与空状态

- 分页参数非法时归一为第一页；超过末页时展示最后有效页。
- 雷达磁盘缓存损坏、版本不匹配或日期不匹配时忽略缓存并实时重建，不影响页面可用性。
- 趋势图数据不足时不补零、不伪造均线。
- 同步失败继续展示错误摘要，移动端不得造成横向页面溢出。

## 测试与验收

- 模板路由测试覆盖全部八类页面：工作台、短线、龙头、长线、市场雷达、单股、数据状态、AI解释。
- 单元测试先验证分页边界、缺失值格式化、内部标签转换、雷达缓存回退和趋势序列生成。
- 模板测试验证移动菜单语义、表单标签、日期控件、分页导航、POST 重新生成按钮和统一表格容器。
- 全量 Python 测试必须通过。
- 部署后验证全部页面 HTTP 200；市场雷达冷启动应优先命中持久化缓存，目标响应时间低于 1 秒；短线和长线页面 HTML 体积相对当前分别下降至少 40%。

## 约束

- 不引入 React、Vue、Bootstrap、Tailwind 或外部 CDN。
- 不增加自动下单、仓位管理或交易执行能力。
- 不修改选股策略阈值和回测逻辑。
- 所有新增代码注释使用中文。
- 不覆盖或提交服务器运行态数据库。

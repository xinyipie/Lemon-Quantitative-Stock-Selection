# 策略研究基础正确性设计

## 目标

在不改变选股因子权重和自动下单边界的前提下，先修正会导致实盘、回测和研究报告不可比较的基础问题：策略身份漂移、波段宽池财务覆盖缺口、历史财务时点泄漏，以及组合层隐含超额敞口和非逐日盯市净值。

## 范围

本轮实现四项：

1. 为短线和波段建立显式策略 profile 契约；官方回测入口默认与实盘配置一致，研究调用仍可显式选择旧 profile。
2. 波段宽池在进入 `select_longterm_pool` 前补齐全部新增股票的批量财务数据。
3. 历史截面财务数据缺少或包含无效 `ann_date` 时 fail-closed，不再静默使用无法证明当时已知的数据。
4. 回测引擎限制跨日最大同时持仓，禁止持有期内同股重复开仓，并用每日收盘价重建盯市净值。

## 非目标

- 不改短线/波段因子权重和准入阈值。
- 不把观察层、新闻或龙虎榜升级为正式信号。
- 不实现自动下单、仓位管理或交易执行系统。
- 本轮不重写涨跌停、停牌复牌和日内 OHLC 成交顺序；这些进入下一批执行模型修复。
- 本轮不承诺历史点时 ST/行业映射完整，先保证财务公告时点和股票上市/退市字段接口可被后续工作使用。

## 设计

### 策略身份

新增集中式 profile 解析函数，返回短线 factor/style/consensus 与波段 profile。命令行的正式回测默认读取 `config.py` 的实盘配置，并把最终 profile 写入日志与结果元数据。研究代码若需要 `original` 或 `zscore_v4_1`，必须显式传入。

### 波段财务覆盖

保留短线阶段已有财务查询。构造 `lt_stocks` 后只对缺失代码做一次批量补充，并合并为 `longterm_financial_dict`。禁止逐股请求。

### 财务时点

统一使用 `_filter_announced_rows(df, trade_date, dataset_name)`：历史日期存在时，要求 `ann_date` 列存在，并只保留格式正确且 `ann_date <= trade_date` 的记录。缺列或全部无效时抛出数据质量异常，由调用方终止该截面策略计算，而不是使用未来或未知数据。

### 组合与净值

短线和波段统一使用最大槽位数。按买入日顺序，仅在有空槽位时接受新信号，并禁止同股重叠。净值按每日收盘盯市：未平仓使用当日收盘，卖出日使用实际卖出价，空槽位视为现金；费用继续使用现有费率。该模型用于替换只在卖出日一次性记收益的曲线。

## 验收

- 官方短线和波段回测入口解析出的 profile 与实盘配置一致。
- 波段宽池新增代码能获得财务数据，不再因使用短线财务字典而系统性丢失。
- 缺失 `ann_date` 的历史财务数据测试必须失败；未来公告不能进入截面。
- 任意日期同时持仓数不超过 `max_positions`，同股不重叠。
- 未平仓价格下跌时，最大回撤在卖出前就反映到净值。
- 现有核心测试和新增测试全部通过。

# 服务器每日完整更新定时任务设计

## 目标

在服务器 `175.25.50.119` 上部署当前 Stock 选股助手，并按服务器时区每天凌晨 02:00 自动执行一次完整更新。

完整更新命令为：

```bash
python daily_web_update.py --mode full
```

本任务只更新选股工具的数据、信号、复盘和审计结果，不增加或执行任何自动下单、仓位管理或交易操作。

## 推荐方案

使用服务器原生定时任务。Linux 服务器使用 cron；若远端实际为 Windows，则使用任务计划程序实现同等语义。原生调度不依赖本地电脑或 Codex 客户端持续在线。

## 部署流程

1. 通过 Codex 已登记的 SSH 主机 `175.25.50.119` 确认操作系统、登录身份、项目目录、Python 解释器和现有调度配置。
2. 将本地 `main` 分支当前提交同步到服务器项目目录，保留服务器环境变量和数据文件。
3. 创建一个服务器端更新包装脚本，切换到项目目录后调用完整更新命令。
4. 配置每天 02:00 运行该包装脚本，并确保同名任务只有一份。
5. 检查定时任务配置和包装脚本的命令解析结果，不主动执行耗时的完整更新。

## 并发和失败处理

- 使用服务器原生文件锁，确保前一次任务未结束时跳过重复启动。
- 每次运行将标准输出和标准错误追加到独立日志文件。
- 更新命令的退出码原样返回，便于系统日志和后续排障识别失败。
- 定时任务使用绝对项目路径和明确的 Python 解释器，避免交互式 shell 与 cron 环境差异。

## 环境和数据保护

- 复用服务器已有的 `TUSHARE_TOKEN`、`DASHSCOPE_API_KEY` 和 Python 依赖环境，不把密钥写入仓库或调度配置。
- 不覆盖服务器的 `data/`、数据库、缓存、日志和未纳入版本控制的环境配置。
- 若服务器项目目录存在未提交代码改动，停止代码同步并报告冲突，不强制覆盖。

## 验收标准

- 服务器项目代码与本地目标提交一致。
- 服务器上仅存在一个对应的每日 02:00 定时任务。
- 调度命令从正确项目目录执行 `daily_web_update.py --mode full`。
- 调度具备互斥锁和持久日志。
- 环境变量和业务数据未被写入仓库或覆盖。

# 全站看板可用性改造设计

## 目标

让所有页面在宽屏、普通桌面和移动端都形成明确的阅读顺序：先结论与操作，再核心数据，最后历史研究与辅助说明。画布使用全部可用宽度，但正文、表格和卡片通过内部网格控制行长和信息密度。

## 设计方向

采用统一的“研究工作台”视觉体系，不改业务逻辑、不删数据、不增加交易执行能力。公共层负责页面间一致性，各页面通过 `body.page-*` 类获得任务导向布局。

## 页面策略

- 工作台：强化今日决策和异常提醒，压低历史解释与辅助模块的视觉权重。
- 短线复盘：筛选区保持清晰，核心表格增强横向扫描、固定关键列并提高行可读性。
- 热门龙头：候选卡优先，研究统计弱化为证据层，三组候选在宽屏并列。
- 长线池：当前状态与生命周期优先，历史样本和审计表形成次级研究区。
- 市场雷达：简报、利多利空、行业与个股证据形成主阅读流，复盘与历史数据作为侧栏或次级区。
- 单股体检：结论与走势优先，诊断证据两列组织，历史信号保持完整。
- 数据状态：异常和更新入口优先，表覆盖保持紧凑。
- AI解释：支持与风险、观察与失效成对展示，事实核对作为次级证据。

## 响应式规则

- `>= 1680px`：充分利用宽屏，页面核心模块使用两列或三列网格。
- `1081px - 1679px`：保持现有桌面布局，避免过度压缩。
- `<= 1080px`：所有核心网格回落为单列，表格允许横向滚动。
- `<= 720px`：按钮、搜索和筛选控件占满可用宽度，保证触控可用性。

## 验收标准

- 所有主页面继续正常返回并保留现有内容。
- 宽屏不出现由页面容器造成的大面积右侧留白。
- 工作台、雷达、龙头和解释页在宽屏形成清晰并列结构。
- 数据表具有稳定表头、明显行反馈和关键列定位能力。
- 移动端不产生页面级横向溢出。

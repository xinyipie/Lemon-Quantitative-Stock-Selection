# 2026-07-09 短线实盘推送历史展示设计

## 背景

当前短线页已经有两类最新卡片：

- 正式强推荐：`source = live`，仍以 v39 强推荐层为准。
- 观察候选：`source = live_observe`，来自 `best_balance` 观察层。

页面仍缺少一个历史视角，用户看不到过去实盘推送过哪些股票，也不方便区分哪些是正式强推荐、哪些只是观察候选。

## 目标

1. 在 `/signals` 短线页展示最近 30 条实盘推送历史。
2. 历史范围只包括真实实盘推送：`source in ("live", "live_observe")` 且 `mode = "short"`。
3. 每条记录必须明确标记层级：
   - `强推荐`：正式 live 推送。
   - `观察候选`：live_observe 推送。
4. 不混入 `backtest_ic_short`，避免把回测样本误读为历史实盘推荐。
5. 不改变现有短线复盘表和胜率统计口径。

## 页面设计

在 `/signals` 页面中，把新模块放在强推荐卡片、观察候选卡片之后，筛选表单之前。

模块标题：`短线实盘推送历史`

展示最近 30 条记录，按 `trade_date desc, rank asc` 排序。每条显示：

- 日期
- 层级标签：强推荐 / 观察候选
- 股票名称与代码
- 行业
- 分数
- 入场时机：T1 / T3 / T5，如无则显示 `-`
- 推荐原因：优先使用结构化观察原因，其次使用原始推荐原因

## 数据流

1. 新增 service 函数读取历史：
   - 输入：`limit = 30`
   - 使用现有 `get_recent_signals`
   - `source = ["live", "live_observe"]`
   - `mode = "short"`
2. 新增轻量格式化函数：
   - 给每条记录补 `history_layer_label`
   - 给每条记录补 `history_reason`
   - 给每条记录补 `history_entry_timing`
3. `/signals` 路由把 `live_push_history` 传入模板。
4. 模板新增一个紧凑表格或列表区块。

## 边界

- 首页暂不展示历史，避免首页过重。
- 不新增数据库表。
- 不写自动交易、仓位、下单相关逻辑。
- 不修改线上强推荐和观察候选生成策略。

## 验证

1. 单元测试：service 能返回 live 与 live_observe 两类历史，并正确标记层级。
2. 单元测试：backtest_ic_short 不出现在实盘推送历史中。
3. 模板渲染检查：`/signals` 可正常返回。
4. 回归测试：现有强推荐卡片、观察候选卡片测试继续通过。

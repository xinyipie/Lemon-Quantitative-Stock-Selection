# 因子中英对照表

这份表用于阅读回测报告、交易诊断和实验日志。以后讨论因子时，默认采用“中文名（英文变量名）”的写法。

| 英文变量名 | 中文名 | 通俗含义 | 读数提醒 |
|---|---|---|---|
| `original_score` | 原始总分 | 回测重排前系统给出的原始评分。 | 高分不一定更好，需结合分组收益验证。 |
| `score_base` | 基础总分 | 进入实验重排前的基础分数。 | 用来判断新排序是否只是复制旧排序。 |
| `short_score` | 短线分数 | 短线选股模块最终用于排序或展示的分数。 | 主要看它与实际收益是否同向。 |
| `longterm_score` | 波段分数 | 波段/中长线逻辑里的评分。 | 当前短线实验里通常只是辅助字段。 |
| `factor_volume_ratio` | 量能质量 | 对量比是否健康的打分，不等同于原始量比。 | 太低没活性，太高可能是冲高风险。 |
| `factor_drawdown` | 回撤位置得分 | 对“从高点回落后的位置”做的评分。 | 分高不一定安全，过深回撤可能是假机会。 |
| `factor_inflow` | 资金流强度 | 主力资金流入或资金面支持程度。 | 如果赢家输家差异很小，说明它不适合单独加权。 |
| `factor_turnover` | 换手活跃度 | 换手率是否处在适合短线交易的区间。 | 过低没人接，过高可能过热。 |
| `factor_sector` | 板块位置/热度 | 板块补涨、共振或热度相关评分。 | 板块太热时可能接近尾声，需要分市场状态看。 |
| `factor_pattern` | 形态质量 | K线结构、均线、突破/修复等形态综合质量。 | 通常越强越好，但仍要防止追高。 |
| `factor_counter_trend` | 反趋势确认 | 对逆势修复、反包、转强迹象的确认。 | 高一点可能代表弱环境中出现了修复信号。 |
| `factor_wyckoff` | Wyckoff结构 | 用 Wyckoff 思路衡量吸筹、震荡、突破结构。 | 不是越高一定越好，要看当前市场风格。 |
| `factor_accel` | 加速度信号 | 涨跌、资金、量能等是否出现加速。 | 加速可能是启动，也可能是短线过热。 |
| `change` | 当日涨跌幅 | 选股日当天的涨跌幅。 | 太高可能买点偏晚，太低可能还没转强。 |
| `volume_ratio` | 量比 | 当日成交量相对平时的放大程度。 | 原始字段，和 `factor_volume_ratio` 不同。 |
| `drawdown_from_high` | 距高点回撤 | 当前价格距离近期高点回落了多少。 | 数值越大代表离高点越远，但不一定更安全。 |
| `turnover` | 换手率 | 当日成交活跃程度。 | 需要看区间，极端值容易失真。 |
| `mfe_pct` | 持仓期最大浮盈 | 买入后持仓窗口内曾经最多赚多少。 | 用来判断票有没有冲高能力。 |
| `mae_pct` | 持仓期最大浮亏 | 买入后持仓窗口内曾经最多亏多少。 | 用来判断买点是否容易先挨打。 |
| `best_close_pct` | 窗口最好收盘收益 | 持仓窗口里按收盘价计算的最好收益。 | 比 MFE 更保守，排除盘中一闪而过。 |
| `worst_close_pct` | 窗口最差收盘收益 | 持仓窗口里按收盘价计算的最差收益。 | 用来看持仓期间的实际压力。 |
| `window_end_pct` | 窗口结束收益 | 持仓观察窗口最后一天的收益。 | 用来判断走势是否能延续。 |
| `market_style` | 市场风格 | 系统判断的行情类型，例如震荡、弱动量。 | 同一因子在不同风格下可能效果相反。 |
| `macro_mode` | 宏观模式 | 周线或大盘级别的风险/进攻状态。 | 用来决定是否需要更谨慎。 |
| `exit_reason` | 卖出原因 | 这笔交易最终为什么卖出。 | 用来区分选股问题和卖点问题。 |

## 当前读法

- 优先关注“赢家和输家差异大”的因子，而不是只看单个股票分数高低。
- 如果某个因子在赢家和输家之间差异很小，它暂时不适合被大幅加权。
- 如果最高分组不是最好，说明总分排序需要校准，但不要直接推翻全部权重。

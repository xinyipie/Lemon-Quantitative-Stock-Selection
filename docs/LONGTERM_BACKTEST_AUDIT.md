# 波段回测审计

更新日期：2026-06-04

## 结论

当前波段回测的组合口径存在重大解释风险：它不是普通的“最多持有 Top3、总仓位约100%”模型。

回测主循环每天都可能开新的 Top3，而旧持仓不会占用后续开仓名额。净值计算又按每笔交易固定 `1 / TopN` 权重计入收益，因此长持仓会叠加出远超 100% 的名义暴露。

这会导致：

- 强牛市收益被大幅放大；
- 弱市和震荡市回撤也被大幅放大；
- 单笔平均收益、MFE、胜率仍有参考价值；
- 组合总收益、最大回撤不能直接当作实盘可执行结果。

## 代码口径

`backtest_v2.py` 当前净值模型：

```text
weight_per_stock = 1.0 / top_n
每笔卖出收益按固定 1/top_n 加权进入 NAV
```

但波段每天都可能新开 Top3，且持仓最长 60 天，所以真实同时持仓数可能远大于 3。

因此一个 40 笔同时持仓的阶段，名义暴露接近：

```text
40 × 1/3 = 13.33 倍
```

这解释了为什么 2025 legacy 可以出现 +325%，同时最大回撤也接近 90%。

## 后续规则必须先定义

在继续波段因子实验前，必须先决定组合层规则：

1. 最大同时持仓数是多少；
2. 同一股票未平仓时是否允许重复买入；
3. 每笔仓位是固定 `1/top_n`，还是按最大持仓数分配；
4. 空仓资金是否闲置；
5. 每日新信号是否只补足空位，而不是每天固定再买 Top3。

推荐先做一个可执行的稳健口径：

```text
longterm_portfolio_v1
- 最大同时持仓 3 或 5 只
- 同一股票未平仓时禁止重复开仓
- 每只仓位 = 1 / max_positions
- 只有空位时才允许新开仓
- 仍保留 T+1 开盘买入、低开跳过、原出场规则
```

## 当前可用结论

在组合口径修正前：

- 不使用波段总收益做策略定板；
- 可以参考单笔交易质量、MFE/MAE、窗口期末收益；
- 可以参考候选是否有大波段爆发力；
- 不继续做波段权重微调，避免在错误组合口径上过拟合。
